## Project in one paragraph

Flutter mobile Sudoku game (iOS + Android) backed by Supabase (Postgres + Auth + Realtime + Edge Functions). Per-puzzle "IQ" score from time + mistakes + difficulty, compared against Einstein's commonly-cited 160. Five tiered leaderboards (Easy / Medium / Hard / Expert / Evil) ranked by personal best IQ per tier. Free + one-time $4.99 Pro IAP (deferred to Phase 6). Brand: neon-circuit cerebral; cyan + violet over deep navy. Single git repo at `/media/Windows1/Projectz/HeffelHoff-Sudoku/`, deployable to a real Galaxy S918B over USB ADB (serial in `.env.claude.local.md`).

## Open follow-ups / known issues

- **`onNext` climb animation passes `previousIq: 0`** — we don't pre-fetch the prior best, so the count-up always starts at 0. Fine on first wins, sub-optimal on returning users. Pre-fetch the existing leaderboard row before submitting the win to fix.
- **No actual position-swap animation** when the user's new IQ overtakes the row above — the row reorders via `AnimatedSwitcher` in the realtime stream re-emit, but there's no smooth row-slide animation between old and new positions. Adding a custom `AnimatedReorderableList` or per-row `AnimatedPositioned` is Phase 4 polish.
- **Buy-hint button no-op** (task #42) — UI button exists; `MockMonetizationService.purchase()` returns true but no consumer reacts. Wait until real IAP is wired in Phase 6.
- **Three info-level analyzer lints** in `widgets/structure_complete_toast.dart` (missing trailing commas) and one each in `tier_card.dart` (prefer_const_constructors) and `pubspec.yaml` (sort_pub_dependencies). Zero errors. Safe to ignore until a cleanup pass.

## Quirks / non-obvious decisions

- **JDK 21 required for Android builds.** The dev machine has JDK 25 as default, but Kotlin (used by the AGP plugin) can't parse the JDK 25 version string and throws `IllegalArgumentException: 25.0.2` during `Settings/TopLevel/stage1` Kotlin DSL compilation. Fix: `flutter config --jdk-dir=/usr/lib/jvm/java-21-openjdk` (already configured on the dev machine).
- **AdMob App ID must be in `AndroidManifest.xml`.** Without it, `MobileAdsInitProvider` crashes the app at startup with `IllegalStateException: Missing application ID`. The literal value lives in `.env.claude.local.md`.
- **Migrations are idempotent** (`CREATE TABLE IF NOT EXISTS`, `DROP POLICY IF EXISTS ... CREATE POLICY`, `DROP TRIGGER IF EXISTS ... CREATE TRIGGER`, `DO $$ ... pg_constraint lookup ... $$`). `supabase db push` is safe to re-run.
- **`NOTIFY pgrst, 'reload schema';`** appears at the end of migrations 0004 and 0005. Without it the PostgREST schema cache stays stale for several minutes after a fresh push and the leaderboard / profiles queries fail with PGRST205 even though the tables exist.
- **Direct FK from `leaderboard_entries.user_id` → `public.profiles(id)`** (migration 0004) is required for the leaderboard's embedded `profiles!inner(...)` join. Both columns reference `auth.users(id)` independently but PostgREST can't traverse through the unexposed `auth` schema.
- **Privacy fix in 0005**: `handle_new_user` no longer seeds `display_name` from the email's local-part (it leaked email fragments on the public leaderboard). Backfills any existing email-derived names to `Player####` placeholders. Users pick a real username via the AccountSheet.
- **Sound files** in `assets/audio/*.wav` are *placeholder synths* (Python stdlib via `tools/synth_sounds.py`). Replace any of them with real audio whenever sourced — `SoundService` reads asset paths, doesn't care about origin.
- **Peer-solve %** is a deterministic synthetic value in [20.00, 40.00] computed from the puzzle seed (`PeerSolveBanner.syntheticPercent(seed)`). Phase 5 wires real `puzzle_stats` aggregation; until then this gives the banner a believable, stable number.
- **`taskAffinity=""` + `singleTop`** previously caused multiple stale `mHiddenTasks` records that bled into Recents. Fixed by switching `MainActivity` to `launchMode="singleTask"` with default (package-name) task affinity. A reboot clears any orphaned hidden tasks left over from the older builds.
- **Auto-anonymous sign-in at startup** (`initSupabase()` in `lib/core/supabase/supabase_client.dart`) — without an active session, RLS on `puzzle_attempts` (`auth.uid() = user_id`) silently rejects inserts, so no row gets written, no `update_leaderboard_on_attempt` trigger fires, and the player never appears on the leaderboard. Calling `signInAnonymously()` on first launch keeps the friction-free "tap-to-play" UX while still satisfying RLS. **Requires the Supabase Anonymous provider to be enabled** in the dashboard (Auth → Providers → Anonymous → On). With it disabled, the call throws `AuthApiException(code: anonymous_provider_disabled)` and `[Supabase] Anonymous sign-in failed: ...` shows in logcat. The app keeps working in read-only-leaderboard mode but no wins are recorded.

## Build, run, deploy

The full incantation including Supabase wiring is in `README.md`. The working pattern on the dev machine:

```bash
export PATH="$HOME/Android/Sdk/platform-tools:$HOME/flutter/bin:$PATH"
flutter analyze                                                   # 0 errors expected
flutter build apk --debug --target-platform android-arm64
adb -s "$DEVICE_SERIAL" install -r build/app/outputs/flutter-apk/app-debug.apk
adb -s "$DEVICE_SERIAL" shell am force-stop "$ANDROID_APP_ID"
adb -s "$DEVICE_SERIAL" shell am start -n "$ANDROID_APP_ID/.MainActivity"
```

`$DEVICE_SERIAL` and `$ANDROID_APP_ID` come from `.env.claude.local.md`.

For Supabase changes:

```bash
supabase db push          # idempotent; safe to re-run
```

If the leaderboard shows "Schema cache not ready", paste in the SQL editor:

```sql
NOTIFY pgrst, 'reload schema';
```

## How to verify the app works end-to-end

After a fresh clone + setup, smoke test:

1. Splash plays, transitions to home in ~2.2s.
2. Tier list shows 5 cards, Evil at the bottom is the most decorated. They evenly fill the viewport on a 6"+ phone.
3. Tap any tier → game loads under 2s. Solve a puzzle (or fail it via 3 wrong placements). Both win and lose post-game screens render with their action buttons tappable in the pinned footer.
4. Sound plays on placement, error, structure complete, digit complete, puzzle complete, genius solve.
5. Tap the avatar → sign up → username editor opens automatically with `Player####` prefilled. Set a real username. Save.
6. Tap the trophy. Leaderboard loads (after migrations are applied). You appear in the list under your chosen username — never under your email.
7. Tap "Next — see leaderboard" on the post-game screen → climb animation runs.
8. Lose all 3 lives → out-of-lives dialog → "Watch ad" plays a (test) rewarded ad and restores 1 life.
9. Complete 3 puzzles → on the 3rd, a (test) interstitial plays before the post-game screen renders.

If any of those break, check `adb -s "$DEVICE_SERIAL" logcat -d -t 250 | grep -iE "FATAL|AndroidRuntime.*E"` — the most common failure modes (AdMob, JDK 25, missing migrations) are documented in this file's "Quirks" section.

Seed data: `supabase/migrations/0006_seed_fake_leaderboard.sql` populates 50 synthetic users with consistent IQ scores (computed via `compute_iq()`) so the leaderboard isn't empty pre-launch. All seeded rows are tagged `raw_app_meta_data->>'provider' = 'hh_seed'`; remove with `delete from auth.users where raw_app_meta_data->>'provider' = 'hh_seed';`. Real users are never touched.

Ad strategy (decided after first device test):

- **Mid-puzzle: no automatic ads.** No banners, no interstitials interrupting gameplay.
- **Out-of-lives dialog: opt-in rewarded ad** — user picks "Watch ad — 1 life", "Buy 1 life $X.XX", or "Give up". User-initiated and capped to once per puzzle.
- **Puzzle completion: throttled interstitial** — `_CompletionInterstitialGate` fires `maybeShowCompletionAd()` in `initState`. `InterstitialAdService` shows on every Nth (default 3rd) completion. Pro will bypass this entirely once IAP is wired.

Naming conventions (for cross-platform clarity):

- **iOS Bundle ID:** hyphenated (Apple Dev Portal forbids underscores). Exact value in `.env.claude.local.md`.
- **Android applicationId / namespace:** single-word, no separator. Exact value in `.env.claude.local.md`. Renamed pre-launch from the old underscore form for visual parity with iOS and conformance to Android conventions.
- **Dart package name** in `pubspec.yaml` is intentionally the older `heffelhoff_sudoku` (snake_case, since Dart package names require snake_case and forbid hyphens). Unrelated to the Android applicationId — kept stable to avoid rewriting every package import in the codebase.
