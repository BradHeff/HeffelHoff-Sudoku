## Project in one paragraph

Flutter mobile Sudoku game (iOS + Android) backed by Supabase (Postgres + Auth + Realtime + Edge Functions). Per-puzzle "IQ" score from time + mistakes + difficulty, compared against Einstein's commonly-cited 160. Five tiered leaderboards (Easy / Medium / Hard / Expert / Evil) ranked by personal best IQ per tier. Free + one-time $4.99 Pro IAP (deferred to Phase 6). Brand: neon-circuit cerebral; cyan + violet over deep navy. Single git repo at `/media/Windows1/Projectz/HeffelHoff-Sudoku/`, deployable to a real Galaxy S918B (`R5CWB08BJCR`) over USB ADB.

## Open follow-ups / known issues

- **`onNext` climb animation passes `previousIq: 0`** — we don't pre-fetch the prior best, so the count-up always starts at 0. Fine on first wins, sub-optimal on returning users. Pre-fetch the existing leaderboard row before submitting the win to fix.
- **No actual position-swap animation** when the user's new IQ overtakes the row above — the row reorders via `AnimatedSwitcher` in the realtime stream re-emit, but there's no smooth row-slide animation between old and new positions. Adding a custom `AnimatedReorderableList` or per-row `AnimatedPositioned` is Phase 4 polish.
- **Three info-level analyzer lints** in `widgets/structure_complete_toast.dart` (missing trailing commas) and one each in `tier_card.dart` (prefer_const_constructors) and `pubspec.yaml` (sort_pub_dependencies). Zero errors. Safe to ignore until a cleanup pass.

## Quirks / non-obvious decisions

- **JDK 21 required for Android builds.** The user's Fedora has JDK 25 as default, but Kotlin (used by the AGP plugin) can't parse the JDK 25 version string and throws `IllegalArgumentException: 25.0.2` during `Settings/TopLevel/stage1` Kotlin DSL compilation. Fix: `flutter config --jdk-dir=/usr/lib/jvm/java-21-openjdk` (already configured on the dev machine).
- **`google_mobile_ads` is *not* in `pubspec.yaml`.** Adding it breaks app startup with `IllegalStateException: Missing application ID` from `MobileAdsInitProvider` in the merged `AndroidManifest.xml`. Phase 6 brings it back together with a real AdMob App ID `<meta-data>` tag.
- **`supabase_flutter` *is* in `pubspec.yaml`** — it has no manifest providers that crash at startup, just SDK code. Safe to keep even before auth ships.
- **Migrations are idempotent** (`CREATE TABLE IF NOT EXISTS`, `DROP POLICY IF EXISTS ... CREATE POLICY`, `DROP TRIGGER IF EXISTS ... CREATE TRIGGER`, `DO $$ ... pg_constraint lookup ... $$`). `supabase db push` is safe to re-run.
- **`NOTIFY pgrst, 'reload schema';`** appears at the end of migrations 0004 and 0005. Without it the PostgREST schema cache stays stale for several minutes after a fresh push and the leaderboard / profiles queries fail with PGRST205 even though the tables exist.
- **Direct FK from `leaderboard_entries.user_id` → `public.profiles(id)`** (migration 0004) is required for the leaderboard's embedded `profiles!inner(...)` join. Both columns reference `auth.users(id)` independently but PostgREST can't traverse through the unexposed `auth` schema.
- **Privacy fix in 0005**: `handle_new_user` no longer seeds `display_name` from the email's local-part (it leaked email fragments on the public leaderboard). Backfills any existing email-derived names to `Player####` placeholders. Users pick a real username via the AccountSheet.
- **Sound files** in `assets/audio/*.wav` are *placeholder synths* (Python stdlib via `tools/synth_sounds.py`). Replace any of them with real audio whenever sourced — `SoundService` reads asset paths, doesn't care about origin.
- **Peer-solve %** is a deterministic synthetic value in [20.00, 40.00] computed from the puzzle seed (`PeerSolveBanner.syntheticPercent(seed)`). Phase 5 wires real `puzzle_stats` aggregation; until then this gives the banner a believable, stable number.
- **Migrations not yet on the user's Supabase**: as of last session, the user has **not yet successfully run `supabase db push` for migrations 0004 + 0005** (0001-0003 ran). They got an FK violation on 0004 (orphaned profile) — fixed in commit `7739091` (0004 now backfills missing profiles before adding FK). Privacy migration 0005 is brand new (commit `534304b`). User needs to run `supabase db push` again to apply both.

## Ad strategy (decided after first device test)

- **Mid-puzzle: no automatic ads.** No banners, no interstitials interrupting gameplay.
- **Out-of-lives dialog: opt-in rewarded ad** — user picks "Watch ad — 1 life", "Buy 1 life $X.XX", or "Give up". User-initiated and capped to once per puzzle. Verified working on device with the test rewarded ad.
- **Puzzle completion: throttled interstitial** — `_CompletionInterstitialGate` wraps both `GameWon` and `GameLost` post-game screens. Fires `maybeShowCompletionAd()` in `initState`. `InterstitialAdService` increments a counter and only actually loads/shows on every Nth (default 3rd) completion. Pro should bypass this entirely once IAP is wired.
- **Two-instance fix:** `AndroidManifest.xml` uses `launchMode="singleTask"` (default `taskAffinity` — package name) + readable `android:label="HeffelHoff Sudoku"` so the app shows as one card in Recents. The earlier `taskAffinity=""` + `singleTop` combo accumulated 10 stale `mHiddenTasks` records that survived reinstalls. Reboot of the device clears `mHiddenTasks`.
- **Apple Team ID (App ID Prefix):** `WWMRM77FM3` — feeds into Sign in with Apple, provisioning profiles, Supabase Auth Apple config, and the eventual `Info.plist` `GADApplicationIdentifier` for iOS AdMob.
- **iOS Bundle ID:** `com.heffelhoff.heffelhoff-sudoku` (hyphenated — Apple Dev Portal rejects underscores).
- **Android applicationId / namespace:** `com.heffelhoff.heffelhoffsudoku` (single word — no underscore, no hyphen). Renamed pre-launch from the old `heffelhoff_sudoku` form for visual parity with iOS and conformance to Android conventions. The Dart package name in `pubspec.yaml` (`heffelhoff_sudoku`) is intentionally unchanged — Dart package names require snake_case and are unrelated to the Android applicationId.
