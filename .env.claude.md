## Cross-machine continuation checklist

What a fresh clone needs to actually build and run:

1. **Clone the repo** + `flutter pub get && dart run build_runner build --delete-conflicting-outputs`
2. **Flutter SDK 3.41.7 (Dart 3.11.5)** + **JDK 21** (`flutter config --jdk-dir=/path/to/jdk-21`) — JDK 25 breaks Kotlin DSL during Gradle stage1. Android SDK 36.
3. **Recreate `.env.claude.local.md`** at the repo root from the prior machine's copy (or paste the relevant IDs from your password manager). It's gitignored, so it doesn't travel via git.
4. **Recreate `android/local.properties`** with the gitignored AdMob keys (the file itself isn't tracked, but Flutter regenerates the `sdk.dir` / `flutter.sdk` lines on first build — append the `ADMOB_*` lines from `.env.claude.local.md`).
5. **Recreate `android/key.properties`** if you have access to the upload keystore — values in `.env.claude.local.md`. Without it, release builds fall back to debug signing (won't be accepted by Play Store).
6. **Restore `android/app/upload-keystore.jks`** from off-machine backup (also in `.env.claude.local.md` instructions). Without this, you cannot ship updates to the same Play listing — the upload key is irreplaceable.
7. **Restore `ios/AuthKey_<KEY_ID>.p8`** if you'll be doing Apple Sign-in JWT regen on this machine (gitignored; back-it-up with the keystore).
8. **Configure Supabase CLI**: `supabase login`, then `supabase link --project-ref <ref-from-local-notes>`. `supabase db push` is safe to re-run (migrations idempotent).
9. **Verify Anonymous provider is enabled** in the Supabase dashboard (Auth → Providers → Anonymous → On). Without this, every fresh-install puzzle win is silently dropped — see "Auto-anonymous sign-in" in Quirks.
10. **Build with the full dart-define set** (see Build section below). The release-mode app needs Supabase URL + anon key + AdMob ad units + AdMob test-device IDs all passed at build time.

## Project in one paragraph

Flutter mobile Sudoku game (iOS + Android) backed by Supabase (Postgres + Auth + Realtime + Edge Functions). Per-puzzle "IQ" score from time + mistakes + difficulty, compared against Einstein's commonly-cited 160. Five tiered leaderboards (Easy / Medium / Hard / Expert / Evil) ranked by personal best IQ per tier. Free + one-time $4.99 Pro IAP (deferred to Phase 6 finish). Brand: neon-circuit cerebral; cyan + violet over deep navy. Single git repo; deployable to a real Galaxy S918B over USB ADB on Android (serial in `.env.claude.local.md`) and to TestFlight via CodeMagic for iOS. iOS device family is **iPhone-only** (`TARGETED_DEVICE_FAMILY = "1"`).

## Quirks / non-obvious decisions

- **JDK 21 required for Android builds.** JDK 25 breaks Kotlin DSL with `IllegalArgumentException: 25.0.2`. Fix: `flutter config --jdk-dir=/usr/lib/jvm/java-21-openjdk`.
- **AdMob App ID must be in `AndroidManifest.xml`.** Without it, `MobileAdsInitProvider` crashes on launch. The literal value is injected at build time via Gradle `manifestPlaceholders` from gitignored `android/local.properties` (key `ADMOB_APP_ID_ANDROID`); manifest source uses `${admobAppId}` placeholder.
- **Migrations are idempotent.** `supabase db push` is safe to re-run.
- **`NOTIFY pgrst, 'reload schema';`** at the end of migrations 0004–0005. Without it the PostgREST schema cache stays stale for several minutes and queries fail with PGRST205.
- **Direct FK from `leaderboard_entries.user_id` → `public.profiles(id)`** (migration 0004) is required for the leaderboard's embedded `profiles!inner(...)` join.
- **Privacy fix in 0005**: `handle_new_user` no longer seeds `display_name` from email local-part; backfills any existing email-derived names to `Player####`.
- **Sound files** in `assets/audio/*.wav` are placeholder synths regenerable via `tools/synth_sounds.py`.
- **Peer-solve %** is deterministic synthetic value in [20.00, 40.00] from puzzle seed. Phase 5 wires real aggregation.
- **`taskAffinity=""` + `singleTop`** previously caused stale `mHiddenTasks`. Fixed via `launchMode="singleTask"` with default (package-name) task affinity. A reboot clears any orphaned hidden tasks from older builds.
- **Auto-anonymous sign-in at startup** (`initSupabase()` in `lib/core/supabase/supabase_client.dart`) — without an active session, RLS on `puzzle_attempts` silently rejects inserts, so no row ever lands and the player never appears on the leaderboard. **Requires the Supabase Anonymous provider toggled ON** in the dashboard (Auth → Providers → Anonymous → Enable). With it disabled, the call throws `AuthApiException(code: anonymous_provider_disabled)`.
- **Session-recovery chain** in `AttemptsRepository._ensureSession()`: existing `currentUser` → `auth.refreshSession()` → `signInAnonymously()` fallback. Catches expired-token edge cases where the UI thinks you're logged in but `currentUser` is null.
- **OAuth deep-link callback** is `heffelhoffsudoku://auth-callback`. Wired into:
  1. `AuthRepository._oauthRedirect` in source
  2. `AndroidManifest.xml` intent-filter on `MainActivity`
  3. `ios/Runner/Info.plist` `CFBundleURLTypes`
  4. **Supabase dashboard's "Redirect URLs" allowlist** (Auth → URL Configuration). All four must agree or the OAuth round-trip fails silently.
- **iOS-only Info.plist requirements** (else App Review or runtime fails):
  - `GADApplicationIdentifier` — without it, `MobileAdsInitProvider` crashes the app on launch
  - `NSUserTrackingUsageDescription` — required when AdMob is present, even if you never call ATT yourself
  - `SKAdNetworkItems` — Apple's iOS 14.5+ install attribution allowlist (44 entries; matches AdMob's published list)
  - `ITSAppUsesNonExemptEncryption=false` — bypasses the export-compliance prompt on every TestFlight upload (the app uses no encryption beyond Apple's URLSession + Keychain frameworks)
- **`PrivacyInfo.xcprivacy`** — required by App Review since May 2024. Lives at `ios/Runner/PrivacyInfo.xcprivacy` and is wired into the Xcode project's Resources build phase via `ios/Runner.xcodeproj/project.pbxproj`. Declares ATT/AdMob tracking + Required Reason API uses (UserDefaults, file timestamp, system boot time, disk space).
- **`Runner.entitlements`** at `ios/Runner/Runner.entitlements` declares Sign in with Apple. Wired into all three Runner build configs via `CODE_SIGN_ENTITLEMENTS` in `project.pbxproj`. Apple Developer Portal must also have the capability enabled on the App ID.
- **iOS device family is iPhone-only** (`TARGETED_DEVICE_FAMILY = "1"`). Set after ITMS-90474 (iPad multitasking orientations) blocked App Store upload. Reverting to Universal requires restoring the four-orientation `UISupportedInterfaceOrientations~ipad` block AND providing iPad screenshots.

## Build, run, deploy

### Android — debug to test phone

```bash
export PATH="$HOME/Android/Sdk/platform-tools:$HOME/flutter/bin:$PATH"
flutter analyze                                                   # 0 errors expected
flutter build apk --debug --target-platform android-arm64 \
  --dart-define=SUPABASE_URL=... \
  --dart-define=SUPABASE_ANON_KEY=... \
  --dart-define=ADMOB_REWARDED_ANDROID=... \
  --dart-define=ADMOB_INTERSTITIAL_ANDROID=... \
  --dart-define=ADMOB_TEST_DEVICE_IDS=...
adb -s "$DEVICE_SERIAL" install -r build/app/outputs/flutter-apk/app-debug.apk
adb -s "$DEVICE_SERIAL" shell am force-stop "$ANDROID_APP_ID"
adb -s "$DEVICE_SERIAL" shell am start -n "$ANDROID_APP_ID/.MainActivity"
```

`$DEVICE_SERIAL`, `$ANDROID_APP_ID`, and the dart-define values come from `.env.claude.local.md`. Test-device ID protects the dev phone from accidental clicks on real ads (AdMob policy enforcement); register the same hash in **AdMob Console → Settings → Test Devices** as a belt-and-braces second layer.

### Android — release for Play Store

Same command with `--release` and a signed AAB:

```bash
flutter build appbundle --release \
  --dart-define=SUPABASE_URL=... \
  --dart-define=SUPABASE_ANON_KEY=... \
  --dart-define=ADMOB_REWARDED_ANDROID=... \
  --dart-define=ADMOB_INTERSTITIAL_ANDROID=... \
  --dart-define=ADMOB_TEST_DEVICE_IDS=...
```

Output: `build/app/outputs/bundle/release/app-release.aab` (~65 MB). Upload to **Play Console → Internal Testing → Create new release**. Signed with `android/app/upload-keystore.jks` via `android/key.properties` (both gitignored — back up off-machine; losing the keystore means no future updates under the same upload key unless Google rotates via Play App Signing support).

### iOS — release via CodeMagic

CodeMagic is configured via the web UI (no `codemagic.yaml` in repo). Setup it expects:

1. **App Store Connect API Key** uploaded to CodeMagic Code Signing Identities. Issued from App Store Connect → Users and Access → Integrations → Keys → role **App Manager**. Different `.p8` than the Sign-in-with-Apple key.
2. **iOS code signing** in workflow: Automatic, profile type **App Store**, API key reference name pointing at the API key from step 1.
3. **Publishing → App Store Connect**: enable, same API key reference, "Submit to TestFlight" (or "Submit to App Store" once ready).
4. **Environment variables / dart-defines** for the iOS build: `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `ADMOB_TEST_DEVICE_IDS`, plus `ADMOB_REWARDED_IOS` and `ADMOB_INTERSTITIAL_IOS` once those units exist.
5. **Distribution cert hygiene**: Apple allows ~1 active iOS Distribution cert per team. If the API auto-create returns 409, revoke unused certs in the Apple Developer Portal AND clear `CERTIFICATE_KEY` env var in CodeMagic so it generates a fresh key + cert pair.

Common upload errors and fixes:
- **ITMS-90161** (Invalid Provisioning Profile / Missing code-signing certificate) — CodeMagic was given a stale `CERTIFICATE_KEY`; revoke orphan certs + clear the env var.
- **ITMS-90474** (iPad orientations) — fixed by `TARGETED_DEVICE_FAMILY = "1"` (iPhone-only).
- **Encryption export compliance prompt** — answered via `ITSAppUsesNonExemptEncryption=false` in Info.plist. Pick "None of the algorithms mentioned above" if it ever surfaces in App Store Connect.

## How to verify the app works end-to-end

After a fresh clone + setup, smoke test on the test device:

1. Splash plays, transitions to home in ~2.2s.
2. **Progression card** under "Pick your difficulty." renders — either CTA ("solve your first puzzle…") if first launch, or animated IQ + Einstein bar + delta otherwise.
3. Tier list shows 5 cards, Evil at the bottom is the most decorated.
4. Tap any tier → game loads under 2s. Solve a puzzle (or fail it via 3 wrong placements). Both win and lose post-game screens render with their action buttons tappable in the pinned footer.
5. **Sync badge** above the action buttons cycles `Syncing result…` → `Result saved to leaderboard`. If you see `Couldn't save result`, paste device logcat — the recovery chain or RLS is misconfigured.
6. Sound plays on placement, error, structure complete, digit complete, puzzle complete, genius solve.
7. Tap the avatar → sign up → username editor opens automatically with `Player####` prefilled.
8. **Google + Apple buttons** below the primary action open the system browser, OAuth round-trips, redirect back via the deep link, session activates. (Requires Supabase Google/Apple providers configured + Redirect-URLs allowlist + Apple Services ID + Google OAuth client IDs — see `docs/MONETIZATION.md` and `.env.claude.local.md` for IDs.)
9. Tap the trophy. Leaderboard loads (after migrations are applied). You appear under your chosen username.
10. Tap "Next — see leaderboard" on the post-game screen → climb animation runs.
11. Lose all 3 lives → out-of-lives dialog → "Watch ad" plays a (test) rewarded ad and restores 1 life.
12. Complete 3 puzzles → on the 3rd, a (test) interstitial plays before the post-game screen renders.

If any of those break, check logs:
- Android: `adb -s "$DEVICE_SERIAL" logcat -d -t 250 | grep -iE "FATAL|AndroidRuntime.*E|Attempts|Supabase\] "`
- iOS: TestFlight crash reports + CodeMagic build log

The most common failure modes (Anonymous provider disabled, missing migrations, JDK 25, missing Info.plist keys, missing AdMob App ID) are covered in this file's "Quirks" section.

Ad strategy (decided after first device test):

- **Mid-puzzle: no automatic ads.** No banners, no interstitials interrupting gameplay.
- **Out-of-lives dialog: opt-in rewarded ad** — user picks "Watch ad — 1 life", "Buy 1 life $X.XX", or "Give up". User-initiated, capped to once per puzzle.
- **Puzzle completion: throttled interstitial** — `_CompletionInterstitialGate` fires `maybeShowCompletionAd()` in `initState`. Shows on every Nth (default 3rd) completion. Pro will bypass entirely once IAP is wired.

Naming conventions (for cross-platform clarity):

- **iOS Bundle ID:** hyphenated (Apple Dev Portal forbids underscores). Exact value in `.env.claude.local.md`.
- **Android applicationId / namespace:** single-word, no separator. Exact value in `.env.claude.local.md`. Renamed pre-launch from the old underscore form.
- **Dart package name** in `pubspec.yaml` is intentionally the older `heffelhoff_sudoku` (snake_case — Dart requires it). Unrelated to the Android applicationId; kept stable to avoid rewriting every package import.
