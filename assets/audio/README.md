# Audio assets

`SoundService` (`lib/core/audio/sound_service.dart`) looks up these filenames at runtime. The repo ships placeholder WAVs generated by `tools/synth_sounds.py` so the game has audio out of the box — replace any of them with sourced production audio whenever you're ready (keep the filenames OR update the enum entries in `SoundEvent` to a different extension).

## Regenerating placeholders

```bash
python3 tools/synth_sounds.py
```

Stdlib only — no Python deps required. Tweak the synth in that script if a tone is too harsh / too soft / too long.

## Required files

| Filename | Trigger | Synth character (placeholder) | Length |
|---|---|---|---|
| `place_correct.wav` | Correct digit placed | Rising 2-note blip C5→E5 | 120ms |
| `place_wrong.wav` | Wrong digit placed (life lost) | Descending square-wave thud E4→C4 | 180ms |
| `structure_complete.wav` | A row, column, or 3×3 box completed | Ascending major arpeggio C5-E5-G5 | 500ms |
| `digit_complete.wav` | All 9 instances of a digit placed | Arpeggio C5-E5-G5-C6 + sparkle | 850ms |
| `puzzle_complete.wav` | Puzzle solved (regular) | C-F-C major chord progression | 1.6s |
| `puzzle_complete_genius.wav` | Puzzle solved **under target time** | Brassy saw-wave fanfare → arpeggio → 3-octave C major chord | 3.4s |
| `combo_double.wav` | Two structures complete on same placement | F-A-C arpeggio (4th up from structure) | 500ms |
| `combo_triple.wav` | Three structures complete on same placement | 5-note ascending C5→E6 sparkle | 700ms |

## Where to source them

Free, no-attribution-required SFX libraries that fit:

- **mixkit.co/free-sound-effects/game** — game-specific, MP3 download
- **kenney.nl/assets/category:Audio** — public domain, sets like "UI Audio" and "Casino Audio"
- **freesound.org** — Creative Commons, search for "click positive", "chime", "fanfare short"
- **zapsplat.com** — free with free account, search for "puzzle solve", "ui correct"

## Sourcing tips

- Keep volumes consistent — peak around -6 dB to leave headroom.
- 44.1 kHz / 128–192 kbps MP3 is plenty.
- The `puzzle_complete_genius.mp3` should feel **clearly** bigger than `puzzle_complete.mp3` — that's the whole point of the under-target reward.
- For `place_correct.mp3` you want something you can hear 100+ times per puzzle without fatigue. Test it on loop for 60s before committing.

## Adding more

If you want extra event sounds (e.g. `pencil_toggle.mp3`, `hint_used.mp3`), add them here, then add a corresponding `play()` call to `SoundService` and invoke from the controller.
